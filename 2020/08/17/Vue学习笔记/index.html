Github使用学习

工作区\=\=\=》暂存区（stage）\=\=\=》仓库HEAD(master)

工作区通过add将文件添加至暂存区，再通过commit提交到仓库。

涉及到兼容性的时候，使用notepad++保存为UTF-8 without BOM编码。

git add将文件添加到暂存区

git commit提交暂存区的文件到仓库

git reset HEAD — \ ：针对git add后，不想add了，即把改变从暂存区撤回（只改变暂存区）

git checkout — \：如果将README.md使用git add加入暂存区了后，又修改了工作区的文件。但我现在不想要这个修改的文件，可以使用checkout把暂存区的文件拿回工作区。

git log查看提交历史，通过git log打印出来的id值，就可以找到对应版本。

回到过去

git reset HEAD~：波浪线表示当前commit之前的一个版本，即上一个版本。

N个波浪线，表示倒退N个版本，也可以写程HEAD~N

git reset —mixed HEAD~

git reset移动了两棵树，分别是仓库和暂存区（相当于撤销add和commit操作）

—soft只改变仓库（相当于撤销commit操作）

—hard三棵树全部改变（相当于全部文件回退到上一次版本）

默认参数为mixed

如果版本太多，懒得数个数，可以使用版本号的前几位：git reset a5b1c0

回滚个别文件：见上面

还可以回到现在（通过版本号），如果直接git reset XXX，在工作区是没有现在的文件的，需要使用hard参数将工作区同步。

git reflog查看所有历史提交ID

版本对比

git diff

git diff 窗口容量有限，显示不完全，使用vim的相关命令即可翻页跳转等操作，还可以输入/进行查找，具体帮助输入h，退出用q。

也可使用git diff XXX XXX，来比较两个版本的差异
