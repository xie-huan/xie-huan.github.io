评价分类结果

发表于 2020-05-30 更新于 2020-06-12 分类于 机器学习 阅读次数：

引言

在学习kNN，逻辑回归等算法时，常用分类准确度来衡量模型的好坏。这种评价方式，在数据分布合理的情况下，是没有问题的。但是在数据极度有偏的情况下，这种评价方式就暴露出它的缺点了。

举一个很直观的例子：我们需要一个癌症预测系统，我们用数据训练出一个模型，输入体检相关的信息，判断患者是否患有癌症。假如预测数据中，有100个测试样本，其中只有一个是患癌样本。假设模型不需要任何预测，直接将所有患者都预测为健康，这样，模型的准确率为99/100=0.99，意味这这个什么都没做的模型，预测的准确率达到了99%！

精准率和召回率

显然，由上面直观的例子可以知道，在数据极度有偏的情况下，只使用分类准确率是远远不够的，由此提出了精准率和召回率。在介绍精准率和召回率之前，先说明几个概念。

混淆矩阵

对于二分类问题，混淆矩阵（Confusion Matrix）是一个2*2的矩阵。如下表格所示：

真实\预测 0 1
0 TN FP
1 FN TP

其中行表示真实值，列表示预测值。表格中的TN表示True Negative，即在一次预测中，真实值为0，其预测值也为0的数量；FP表示False Positive，即真实值为0，预测却为1的数量；FN表示False Negative，即真实值为1，预测值却为0的数量；TP表示True Positive，即真实值为1，预测值也为1的数量。显然，TN与TP的和等于真实值中为0的数量，FN与TP的和等于真实值为1的数量。

精准率和召回率

在混淆矩阵的概念下，提出精准率和召回率。

精准率定义：precision = TP / （TP + FP）在预测数据为1的数据上，预测准确率；

召回率定义 ：recall = TP / (TP + FN) 在真实数据为1的数据上，预测的准确率 。

下面举一个简单的例子

真实\预测 0 1
0 9978 12
1 2 8

上表中，计算精准率为precision=8/(12+8)=40%，召回率为recall=8/(8+2)=80%。

在网上，有一张更精确的图可以可视化精准率和召回率的意义：

意义

同样的，我们举一个简单的例子直观地说明精准率和召回率比分类准确度更有意义。在10000个人中，由9990个人是健康的，10个人是患癌的。模型预测所有人都是健康的，则TN=9990，FN=10，TP=0，FP=0。计算其精准率为：nan（因为分母是0），召回率为0。这就证明了在及其有偏的数据中，精准率和召回率比分类准确度更有意义。

解读精准率和召回率

上面用直观的例子说明了精准率和召回率指标相对于分类准确度在某些情况下更有意义，那么具体而言，如何对其数据大小进行解读呢？它们之间的关系又是怎样的呢？

对于不同的场景，有不同的标准。

例如，股票涨跌预测的情况，精准率更为重要。因为股票实际为涨，我们预测为跌，也不会亏损，但是股票实际跌，我们预测了涨，就会导致亏损。所以真正例数量占预测为1的数量的比值就是关注的重点。

再如，癌症预测的情况，召回率更为重要。因为实际患癌症的（FN+TP），预测成为不患癌症（FN），可能就耽误了患者的最佳医治时间，但是若不患癌症，预测为患癌症，也可以通过后续的筛查进一步判断患者是否患癌症。所以在这个场景中，召回率是需要关注的重点。
