计算机组成

综述

计算机系统的三大总线

数据总线、地址总线、控制总线

计算机的主要结构组件

  • CPU
    • 控制单元
    • 算术逻辑单元
    • 寄存器
    • CPU内部互连
  • 主存储器
  • 输入和输出
  • 系统互连

存储器层次结构的原理

局部性原理：处理器倾向于成块地访问存储器中的指令和数据

冯洛伊曼结构的概念

三条主要内容：

  • 数据和指令存储在单一的存储器中
  • 存储器的内容通过位置寻址，而不关心存储在其中的数据类型
  • 以顺序的形式从一条指令到下一条指令执行

数据单元的存取方法

  • 顺序存取：磁带
  • 直接存取：磁盘
  • 随机存取：主存
  • 关联存取：cache

缓存

Cache的映射功能

  • 直接映射
  • 全相联映射
  • 组相联映射

Cache的替换算法

  • LRU：最近最少使用
  • LFU：最近最不经常使用
  • FIFO：先进先出
  • 随便替换：这个技术性能上，还不错。

Cache的写策略

  • 写回法
  • 写直达

I/O

I/O编址方式

  • 存储器映射式
  • 分离式

I/O中断设计问题

  • 软件轮询：费时，但成本低
  • 硬件轮询：菊花链电路
  • 总线仲裁

DMA技术

DMA：直接存储器存取Direct Memeory Access

直观上说：就是CPU将读取数据相关的信息发给DMA，然后DMA接手传输数据，传输完成后，给CPU一个中断信号，CPU开始处理。总结来说，就是CPU只在数据传输的开始和结束时参与。

CPU

CPU的任务

  • 取指令
  • 解释指令
  • 取数据
  • 处理数据
  • 写数据

流水线

基于CPU的指令周期：取值、执行

指令集

寻址方式

  • 立即数寻址
  • 直接寻址
  • 间接寻址
  • 寄存器寻址
  • 寄存器简介寻址
  • 偏移寻址
    • 相对寻址
    • 基址寄存器寻址
    • 变址

指令级并行性和超标量处理器

指令级并行性指：程序中的指令可以并行执行

超标量处理器：使用多条相互独立的流水线的处理器

数据结构与算法

综述

算法的特点

确定性、有穷性、可行性、0个或多个输入、1个或多个输出

逻辑结构和物理结构

逻辑结构：

  • 线性结构
  • 树形结构
  • 图形结构
  • 集合

物理结构

  • 顺序存储
  • 链式存储
  • 索引
  • 散列

线性表

数组和链表的区别

  1. 数组固定长度，链表长度可变
  2. 数组可通过下标随机访问，链表需要根据下标对链表进行遍历
  3. 数组插入操作不方便

头指针和头节点

头指针具有标识作用，头节点是虚拟节点。

栈和队列

栈和队列的应用

栈的应用：

  • 括号匹配
  • 表达式求值
  • 递归程序调用

队列的应用：

  • 在图的层次遍历中使用
  • 在计算机的各种设计中使用：CPU按照队列来处理各个程序的请求

树

Huffman树

暂无

图

邻接矩阵和邻接表

邻接矩阵更适合存储稠密图，邻接表更适合存储稀疏图

最小生成树

Prim算法：选择点

Kruskal算法：选择边

最短路径

Dijkstra算法：单元最短路径，不适合带有负权值的边的图。

Floyd算法：虽然允许负权边，但是不允许有包含带负权值的边组成的回路。

查找

分类

静态查找：顺序查找、二分查找、分块查找

动态查找：二叉查找树、平衡二叉查找树

散列表

概念：根据关键字直接进行访问数据的数据结构

散列函数：

  • 直接定址法：H(key) = a * key + b
  • 除留余数法：H(key) = key % p
  • 数字分析法
  • 平方取中法
  • 折叠法

处理冲突方法：

  • 开放定址法
    • 线性探测法：可能引起聚集
    • 平方探测法：可避免聚集
    • 再散列法
    • 伪随机序列法
  • 拉链法

操作系统

操作系统的特征

并发、共享、异步、虚拟

进程的五个状态

创建、就绪、运行、中断、停止

什么是线程、进程，它们的区别是什么

  • 进程是系统进行资源分配和调度的基本单位，线程是CPU调度和分派的基本单位
  • 线程依赖于进程而存在，一个进程至少有一个线程
  • 进程有自己的独立地址空间，线程共享所属进程的地址空间
  • 线程之间的通信更方便，同一进程下线程共享全局变量等数据，而进程之间的通信需要以以IPC（进程间通信）方式进行

临界区的概念

各个进程中对临界资源进行操作的程序片段
